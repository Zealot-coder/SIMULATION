# Backup/Restore Evidence Folder

Store monthly and incident-related evidence files here.

Naming convention:

`YYYY-MM-DD-supabase-backup-check.md`

Typical contents:

1. Supabase backup/PITR screenshots checklist.
2. Links to GitHub Actions run IDs for:
   - `db-backup`
   - `db-restore-test`
3. Restored backup ID and verification summary.
4. Any exceptions, failures, and remediation notes.
